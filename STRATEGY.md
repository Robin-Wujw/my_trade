# 选股与每日分析策略规范

## 1. 输出边界

系统按不同方法分别筛选，不用一条公式代替全部判断，也不为凑数量降低条件。完整日报固定包含四部分：

1. 基本价值线或附近；
2. 正常基本面选股；
3. 最近 21 个交易日 Formula33 市场结构；
4. 主流板块判断。

价值、基本面、右侧位置和市场结构是不同维度。阶段性下跌不是自动淘汰理由，近期涨幅较高也不是自动入选或淘汰理由。

## 2. Formula33 正式口径

### 2.1 技术公式

前复权日线按交易日计算：

```text
KD = (CLOSE - LLV(LOW, 9)) / (HHV(HIGH, 9) - LLV(LOW, 9)) * 100
K = SMA(KD, 3, 1)
WR10 = 100 * (HHV(HIGH, 10) - CLOSE) / (HHV(HIGH, 10) - LLV(LOW, 10))
WR20 = 100 * (HHV(HIGH, 20) - CLOSE) / (HHV(HIGH, 20) - LLV(LOW, 20))
RSI9 = SMA(MAX(CLOSE - REF(CLOSE, 1), 0), 9, 1)
       / SMA(ABS(CLOSE - REF(CLOSE, 1)), 9, 1) * 100

BASE = K > 80
       AND WR10 < 20
       AND WR20 < 20
       AND RSI9 > 70
       AND 截止观察日上市超过 300 天

XG = 最近 5 个交易日 BASE 每日都成立
```

日报只报告最近21个交易日三浪三命中股票去重数。总市值大于 100 亿元不属于正式技术 XG 的基础条件，只生成一个独立参考池，不能反向改变正式名单。

### 2.2 观察日资格

窗口内曾技术命中，不等于观察日仍可进入正式名单。每只命中股票在本次观察日重新分类：

- `traded`：观察日存在有效交易，可以进入正式 21 日去重名单；
- `suspended_or_no_trade`：观察日停牌或无交易，保留在技术全量和诊断页，但必须从正式名单排除；
- `data_unavailable`：行情请求失败或历史不足，不能伪装成停牌，也不能进入正式名单。

观察日资格每次重新判断，不建立永久排除名单。股票复牌后按新观察日正常计算。

以下数量只保留在 Excel 和完成清单中用于审计，不进入日报正文：

- **技术全量**：上市天数过滤后，窗口内 XG 命中的去重数；
- **观察日无交易技术命中**：技术全量中观察日没有交易的数量；
- **正式结果**：技术全量扣除观察日无交易和数据不可用后的去重数；
- **市值大于 100 亿元池**：同时满足独立市值条件的可交易技术去重数。

### 2.3 固定回归锚点

2026-06-11 至 2026-07-10 共 21 个交易日，固定验收结果如下：

| 项目 | 数量 |
|---|---:|
| 技术条件原始去重 | 193 |
| 上市不足 300 天排除 | 2 |
| 上市过滤后技术全量 | 191 |
| 观察日无交易排除 | 3 |
| 正式结果 | 188 |
| 市值大于 100 亿元独立池 | 145 |

188 只正式结果必须与截图回归夹具逐代码一致。前复权精度锚点为：`001331` 在 2026-05-27 的收盘价是 `48.08`。修改复权、日期窗口、上市天数、观察日状态或去重逻辑时，不得只核对总数。

### 2.4 右侧阶段

Formula33 个股筛选仍严格使用 2.1 节公式，本节只定义最近 21 日正式去重数量的市场阶段，不改变 XG 或 188 名单。每个交易日都独立作为一个观察节点：

```text
N(d) = 在观察日 d 回看最近 21 个交易日，Formula33 正式命中股票的去重数
ΔN(d) = N(d) - N(d-1)
```

只有 `ΔN(d)` 的正负用于累计 21 日节点的连续上升或连续下降日数。这里比较的是相邻两个 21 日正式去重数，不是单日 XG 数量；单日 XG 数量、单日 XG 增减以及它们的连续变化均不参与任何右侧阶段判断。系统逐日处理每一个 21 日节点，并按以下规则维护阶段：

- 连续 3 日正趋势：进入“观察、可右侧积极做多阶段”；
- 连续 5 日正趋势：从第 5 日起“明确进入右侧阶段”；
- 已进入后出现 1 至 4 日负趋势：阶段保持不变，不明确退出；
- 连续 5 日负趋势：从第 5 日起“明确退出右侧阶段”。

阶段状态跨日持久化。日报展示最近 21 个交易日正式去重数量和当前阶段，不展示技术全量、观察日无交易诊断、市值独立池或趋势斜率。

## 3. 基本价值线

### 3.1 公式

```text
基本价值线 = 最新报告期每股净资产
           + 最新年报扣非每股收益
             * (1 + 最新报告期扣非每股收益同比增速)
             * 10
```

- 优先使用扣非每股收益，避免一次性损益被外推。
- 扣非 EPS 同比缺失时，才用同报告期扣非净利润同比近似。
- 年报与最新报告期之间发生送转股时，年报扣非 EPS 必须转成可比口径。
- 财报期、价格截止日和观察日必须一致，不能使用观察日之后的数据。

### 3.2 适用性

公式不是所有股票的统一目标价。自动结果必须保留适用方法和理由；行业分类只能决定优先尝试价值线、PE、PB 或右侧方法，不能单独证明价值线适用。

价值线更适合主业清晰、扣非盈利为正、增长有持续性的公司。自动筛选统一要求观察日总市值不低于 100 亿元，不再使用行业同业数量、市值分位或“龙头代理”作为入选条件。强周期、特殊资产负债表、轻资产强预期、扣非亏损、低基数异常增长或一次性收益主导的公司，应转入历史 PE、历史 PB 或右侧观察。

基本价值线或附近栏目的自动硬条件为：价值线方法适用、观察日总市值不低于 100 亿元、扣非同比非负、质量分不低于 50，且现价/基本价值线不高于 1.08。

```text
现价 / 基本价值线 <= 1.00：价值线内
1.00 < 现价 / 基本价值线 <= 1.08：价值线附近
现价 / 基本价值线 > 1.08：不进入价值线或附近栏目
```

价值线内不等于右侧已经确认，两者必须分开报告。

## 4. 正常基本面选股

生产默认硬条件为：

- 质量分不低于 70；
- 流动性分不低于 55；
- 总市值不低于 100 亿元；
- 扣非同比增长不低于 10%。

在硬条件上再结合主流板块和下跌波段修复分层：

1. 主流板块且修复达到 50%：右侧确认；
2. 主流板块、尚未达到 50%，但长期价格和均量扣抵有利：右侧酝酿；
3. 非主流板块但修复达到 50%：右侧观察；
4. 非主流板块、质量不低于 85、扣非增长不低于 25%，且长期扣抵有利：长期结构观察。

不满足可执行层的股票不能通过调低阈值补足数量。

## 5. 下跌波段与右侧位置

使用观察日及以前最多 500 个交易日，覆盖完整上涨起点。价格从最近上涨高点回撤至少 12% 后才建立有效回调；回调期间创新低就更新低点，重新突破前高后旧回调立即结束，等待下一段有效回调：

```text
区间 = 前高 - 后低
50%位置 = 后低 + 区间 * 0.50
62.5%位置 = 后低 + 区间 * 0.625
75%位置 = 后低 + 区间 * 0.75
原始修复进度 = (现价 - 后低) / 区间
波段分位 = min(100%, max(0%, 原始修复进度))
突破前高幅度 = max(0%, 现价 / 前高 - 1)
```

- 低于 50%：右侧尚未确认；
- 50% 至 62.5%：右侧启动区；
- 达到 62.5%：进一步确认；
- 达到 75%：强修复，同时提示追高和回撤风险。
- 上涨波段 50% = 上涨起点 + (上涨高点 - 上涨起点) × 50%，用于观察上涨结构支撑；
- 回调波段 50% = 回调低点 + (前高 - 回调低点) × 50%，回调阶段只有重新站上该价位才具有右侧突破意义；
- 突破前高后旧回调完成，不继续沿用旧锚点计算数百个百分点的修复进度。

波段锚点和均线、均量、扣抵只能使用观察日及以前数据。

## 6. 板块主线

板块统计和板块观察是两个独立生产步骤。主线评分依据当前实现中的短中期涨幅、成交额比例、大盘弱时韧性、大盘强时进攻性和涨停扩散。

- 板块历史必须达到规定行数和覆盖率；
- 板块实际数据日期必须显示，过期数据不能静默参与结论；
- 主流板块成分必须完整获取，缺失板块代码或成分覆盖失败时该步骤失败；
- 个股属于主流板块只是排序和分层依据，不能绕过基本面硬条件。

## 7. 数据源、缓存和续跑

Formula33 生产固定使用 AkShare 股票清单、交易日和前复权日线。普通 A 股优先使用东方财富前复权端点，失败时可整只股票回退新浪，但不得把两个提供方的历史价格拼成一条复权序列。板块步骤使用其配置的真实板块提供方，并记录来源和覆盖率。

前复权日线缓存版本为 `qfq-cache-v2`，按股票保存并写入 DuckDB：

- 首次运行抓取计算所需历史；
- 后续运行通常只补新增交易日；重叠日 OHLC 改变时视为复权锚点变化，必须完整刷新并原子替换窗口；
- 中途取消后重跑直接复用已完成股票缓存；
- 完成清单只有在全部必要股票处理完成、无可重试数据缺失且输出落盘后写入；
- 同一观察日和相同有效参数再次运行时可直接命中完成清单；
- 周末复跑最近交易日时必须做到 `network_fetch=0`。

不得因为接口失败清空已有有效缓存，也不得把网络错误登记为停牌。

## 8. 日报与推送门禁

日报只使用本轮捕获的 Formula33、板块观察和基本面选股产物，不按文件修改时间猜测任意旧文件。三类输入必须属于同一观察日。

前两组选股与上一交易日最终名单比较，分别记录新进入、退出和分区变化；没有历史基线时明确标记首次建立基线。

完整 HTML 和 CSV 保存到 `var/exports/`。PushPlus 按字符上限自动分页，发送前逐页校验长度，并在全部分页合并后校验应展示的股票代码。生产流水线只有在 Formula33、板块统计、板块观察、因子选股、基本面更新和基本面筛选六个上游步骤全部成功后才进入日报；任何必需输入、日期、覆盖率或分页发送失败都不得视为正式推送成功。

前两组选股额外计算盘中高低点 KD、收盘 KD 开口、RSI999、MACD、ENE、WR、BIAS 和 5/10 日上涨基准量。`|K-D|≥20` 只提示短线收敛倾向；背离是警讯，不单独作为卖出信号。PushPlus 按可操作性降序、风险升序展示，原始指标保留在 CSV。

系统独立保留约 45 个交易日的选股历史，形成两个月突破追踪池：回调修复 45%～50% 强提醒，50%～60% 持续跟踪并累计重新上穿 50% 的次数，超过 60% 停止提醒；突破前高且脱离当前全部筛选后才剔除追踪记录。

## 9. 回测限制

- 候选池、财报、行情和板块只能使用观察日当时可见的数据；
- 波段锚点和指标不能先用完整未来序列计算后再截断；
- 名单冻结后才能计算未来收益；
- 当前文件财务缓存没有完整公告修订版本历史，缺少可信披露时间的样本不能声称是严格时点回测；
- 必须报告样本数、均值、中位数、胜率、最大回撤、缺失数据和交易成本假设。
