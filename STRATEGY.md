# 选股与每日分析策略规范

## 1. 总原则

系统不是一条公式选遍所有股票，而是先判断方法是否适用，再进入不同策略分区。

固定输出四个互相独立的部分：

1. **基本价值线左侧池**：基本价值线确实适用，且现价位于价值线内或附近。
2. **主线成长右侧池**：产业与业绩成立，价格完成下跌趋势50%或62.5%修复。
3. **其他估值池**：不适用基本价值线的公司，使用历史PE、PB或仅作右侧观察。
4. **市场结构层**：三浪三、板块主线、均线均量和量价扣抵，用于确认环境与排序。

不得为了凑满固定数量，按20日或60日涨幅补齐。阶段性下跌本身不是淘汰理由；涨幅较高本身也不是淘汰理由。估值、基本面和右侧位置必须分别说明。

## 2. 基本价值线

### 2.1 含义

基本价值线不是绝对最低价，也不是所有股票的统一目标价。它表示在非熊市环境下，机构可能开始分批建立左侧仓位的估值区间。

### 2.2 公式

```text
基本价值线 = 最新报告期每股净资产
           + 最新年报扣非每股收益
             * (1 + 最新报告期扣非每股收益同比增速)
             * 10
```

数据要求：

- 使用扣非每股收益，避免一次性损益被外推十年。
- 扣非EPS同比缺失时，才允许用同报告期扣非净利润同比近似。
- 送转股发生在年报与最新报告期之间时，扣非EPS必须转为可比口径。
- 公式使用原始同比；质量评分可对极端同比截断，二者不能混用。
- 财报期、披露日和行情截止日必须匹配，历史回测不得使用未来财报。

### 2.3 适用条件

基本价值线是否适用，首先是逐股判断，不是只看行业名称。必须同时满足：

- 产业趋势向上，未来扣非利润和净资产具有持续性。
- 主业清晰，利润不是主要依赖周期价格、资产处置或一次性项目。
- 最新年报扣非EPS为正，最新报告期扣非增长具有可解释性。
- 规模足够，默认总市值不低于100亿元。
- 是行业龙头或细分行业龙头，具有明确行业地位。
- 财务可信度较高，可与产业链上下游财报交叉验证。
- 有机构覆盖和资金聚集基础，具备成为或维持市场主流的可能。
- 当前不是明确熊市中产业与估值同时收缩的阶段。

重资产制造业、科技硬件、通信设备、电子、高端制造、汽车、部分医药和消费公司都**可能**适用，但必须满足上述条件；不能因为属于这些行业就自动套公式。

### 2.4 不适用或谨慎使用

以下情况不能直接使用基本价值线：

- 行业景气明显向下，未来扣非或净资产可能持续下降。
- 强周期资源、公用事业、地产、券商等利润主要随周期波动。
- 银行、保险等资产负债表结构特殊的金融企业。
- 软件、互联网、游戏等轻资产强预期企业，净资产和历史EPS难以刻画其交易逻辑。
- 规模小、行业地位弱、财务可信度不足的公司。
- 扣非利润为负、低基数异常增长或一次性因素主导。
- 业绩起伏巨大，无法合理外推未来十年。

这些股票应转入历史PE/PB、同增速年份比较、当年交易区间或纯右侧策略，不得强套基本价值线。

### 2.5 基本价值线分区

```text
现价 / 基本价值线 <= 1.00：价值线内，进入左侧价值池
1.00 < 比值 <= 1.08：价值线附近观察
比值 > 1.08：不属于价值线左侧池
```

价值线内不要求当日已经右侧转强。每只股票仍需同时展示下跌波段50%和62.5%位置，区分“左侧低估”和“右侧已确认”。

## 3. 替代估值方法

### 3.1 历史PE

适用于利润相对稳定、PE可比较的公司。低估参考为最近十年每年最低PE的异常值剔除均值，或历史分位不高于15%。

### 3.2 历史PB

适用于重资产、周期资源、公用事业、地产、券商、电信运营、建筑、交运等更看重资产或周期位置的公司。使用最近十年低位PB及历史分位。

### 3.3 同增速年份与交易区间

当公司盈利波动大、基本价值线失真时：

- 找历史上与当前扣非增速接近的年份。
- 比较这些年份的低估PE/PB和利润兑现情况。
- 必要时使用当年二级市场已经形成的估值区间。
- 该方法必须说明可比年份和选择理由，不能只给一个结果。

## 4. 下跌趋势右侧50%与62.5%

### 4.1 波段定义

使用截止当日以前的K线，在最近240个交易日中寻找“前高在前、后低在后”的主要下跌波段。不得把低点到后续高点的上涨波段误当成下跌趋势。

```text
区间 = 前高 - 后低
50%位置 = 后低 + 区间 * 0.50
62.5%位置 = 后低 + 区间 * 0.625
75%位置 = 后低 + 区间 * 0.75
当前位置 = (现价 - 后低) / 区间
```

### 4.2 状态

- **低于50%**：右侧尚未确认，可保留在基本价值线左侧池或观察池。
- **50%至62.5%**：右侧启动区。
- **达到62.5%**：右侧进一步确认。
- **达到75%以上**：强修复区，需要同时提示追高和回撤风险，不自动剔除。

“穿越”用于动态买点：前一交易日低于该位置、当前收盘达到或超过该位置。若财报截止日已经位于该位置上方，应标为“截止日已确认”，不能伪装成次日刚穿越。

### 4.3 与基本价值线的关系

基本价值线和右侧波段是两个维度：

| 基本价值线 | 波段状态 | 策略解释 |
|---|---|---|
| 价值线内 | 低于50% | 左侧价值池，等待右侧确认 |
| 价值线内 | 50%以上 | 价值与右侧共振，优先级提高 |
| 价值线上方 | 50%以上 | 主线成长右侧池，估值作为风险说明 |
| 价值线上方 | 低于50% | 暂不属于可执行右侧，继续观察 |

不得用其中一个条件替代另一个。

## 5. 主线成长右侧池

进入该池至少需要：

- 当前属于经过板块数据确认的主流方向。
- 财报扣非增长为正；高增长核心参考不低于25%。
- 质量分参考不低于80，流动性分参考不低于55。
- 总市值参考不低于100亿元。
- 完成下跌波段50%右侧确认；62.5%作为更强确认。

AI/CPO只是历史验证样本，不是永久硬编码特权。任何板块只有在当期成交额、量能、强弱、扩散和业绩共同成立时，才能成为主线成长层。

已经大涨的股票不能仅因涨幅高被删除，但必须报告：现价/基本价值线、50%/62.5%/75%位置、当前位置、均线均量状态和回撤风险。

正常基本面选股按以下层级输出：

1. **主流板块基本面优秀·右侧确认**：业绩硬条件通过，属于当前主流板块，未完整空头排列，且下跌波段修复达到50%。
2. **主流板块基本面优秀·右侧酝酿**：尚未达到50%，但60/120/240日价格扣抵与均量扣抵至少各通过2项，且未完整空头排列。
3. **非主流板块基本面优秀·右侧观察**：不属于当前主流板块，但业绩优秀并已达到50%，作为观察而非主攻方向。
4. **非主流板块基本面优秀·长期结构观察**：质量不低于85、扣非增长不低于25%，长期价格和均量扣抵有利，且未完整空头排列。

业绩硬条件默认包括：质量分不低于70、流动性分不低于55、总市值不低于100亿元、扣非同比增长不低于10%。不得为凑满数量降低硬条件。

## 6. 均线、均量与扣抵

检查5、10、20、60、120、240日均线和均量：

- 当前价格高于对应周期扣抵价，该周期均线具有继续向上条件。
- 当前成交量高于对应周期扣抵量，该周期均量具有继续向上条件。
- 价格和成交量同时满足时，标记该周期量价扣抵共振。
- 长周期金叉确认大趋势时，短线仍可能反向回撤；长周期死叉时，短线也可能反弹。

均线均量用于确认和风险管理，不用于按短期涨幅凑名单。

## 7. 三浪三XG

通达信33公式按最近21个交易日统计，正式口径包含：

- KDJ的K值高于80。
- WR10和WR20都低于20。
- RSI9高于70。
- 总市值大于100亿元。
- 上市超过300天。
- 最近5个交易日每天都满足基础条件。

日报必须同时报告“当日命中数”和“最近21个交易日去重数”。XG是强势结构标签，不替代基本面、价值线或右侧波段判断。

## 8. 板块主线

板块主线依据：

- 3/5/20日涨幅确认趋势。
- 5/20日和20/60日成交额比例确认资金持续性。
- 大盘弱时的抗跌或提前修复。
- 大盘强时的进攻强度。
- 涨停数量和成分股扩散。

板块数据超过7天视为过期，不参与当日结论。个股属于板块不等于自动入选，仍需经过对应策略池条件。

## 9. 每日输出格式

日报固定为以下四个主栏目，其他技术字段只能作为栏目内依据，不能另起一套选股结果：

1. 基本价值线或附近：通过适用性检查且现价/价值线不高于1.08的股票全量输出，不截Top。
2. 正常基本面选股：按财报时点的质量、增长、流动性、估值和行业地位筛选，附50%/62.5%位置。
3. 三浪三上行数量变化：报告当日数量、较前日变化和连续扩张/收缩，据此明确“可以、谨慎或暂停右侧”。
4. 主流板块判断：报告板块数据日期、趋势、量能、强弱和涨停扩散依据。

PE/PB、均线均量、扣抵、风险和数据缺失作为前两栏的说明字段输出。

每只股票至少输出：

```text
代码、名称、策略分区、基本价值线是否适用及理由、现价、基本价值线、
现价/价值线、前高、后低、50%位置、62.5%位置、75%位置、当前位置、
财报期、扣非同比、质量、流动性、主流板块、三浪三状态、均线均量状态、风险。
```

## 10. 历史回测

- 候选池必须按当时财报截止日重建。
- 基本价值线只使用当时已经披露的数据。
- 波段锚点只使用买点当日以前的K线。
- 基本价值线左侧池按截止日价格评估，不强制等待50%。
- 主线成长右侧池逐日等待50%穿越，62.5%记录确认日期。
- 两个池分别统计收益，不得混成一个平均值。
- 不得按最终收益反向挑选股票或调参。
- 必须报告样本数、均值、中位数、胜率、最大回撤、缺失数据和交易成本假设。

## 11. 数据源与持久化

### 11.1 时点审计与动态重建

- 财报期必须在形成日已经依法可见；形成日早于法定披露截止日时禁止运行。
- 所有价格、波段、均线、均量和市值筛选只能使用形成日及以前数据。
- 未来价格只能在名单完全冻结后用于计算收益，不能进入筛选、分层或排序。
- 历史板块行情与历史板块成分必须分别具有时点来源；用当前成分回填历史一律标为不安全并由严格模式阻断。
- 财务缓存应保存首次披露时间、抓取时间和修订版本；缺少修订历史时必须标记为警告。
- 正式选股采用动态重建：每个交易日只能使用当日已经可见的最新财报和当日及以前行情，重新计算价值线距离、质量、流动性、市值、主流板块和技术结构。
- 新财报披露、股票跌入价值线附近或多因素状态改善时，允许动态进入；条件失效时允许动态退出。
- 固定池仅作为归因和对照研究工具，不得替代正式动态选股口径。
- 历史回测必须逐个历史观察日执行同样的动态重建，不能用今天的动态名单回填过去。
- 生产动态截面必须由全市场清单、对应报告期财务缓存和截止当日行情直接生成，不得使用历史回测结果充当候选池。
- 迁移期行情覆盖率低于90%或财务覆盖率低于35%时禁止发布正式日报；任一覆盖率未达到95%时必须标记为覆盖警告。达到稳定生产后再把硬门槛提升到95%。
- 基本价值线自动化适用性至少要求行业已知、同业样本不少于5只且市值位于行业前20%；这是可审计的龙头代理规则，不等同于对真实产业地位的最终证明。
- 财务和行业数据更新必须支持断点续跑，失败股票不得静默丢弃。

- 行情优先AkShare前复权日线，按股票持久化，增量补齐日期。
- 财务数据按股票和报告期持久化，保留来源字段。
- 网络请求低并发、退避重试；失败后从断点继续，不能每天从第一天重抓。
- 任何回退数据源都要在结果中标记，不能静默改变口径。
- 所有生产接口固定按 `AkShare → Baostock → 本地缓存` 顺序调用；AkShare已成功时禁止同时请求Baostock。第三方接口不得插入AkShare与Baostock之间。
- 股票清单、行业、行情和财务分别保存实际 `data_source`，回退原因必须进入运行日志和快照元数据。
- PushPlus只发送摘要，完整结果保存为HTML和CSV，正文不超过配置上限。

## 12. 实现约束

代码中的行业分类只能决定优先尝试哪种估值方法，不能直接证明基本价值线适用。最终必须保留逐股适用性检查及理由。没有足够信息证明适用时，标为“待核验”，不得自动进入基本价值线左侧池。
