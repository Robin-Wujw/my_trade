# 右侧专业量化选股漏斗说明

本文只说明右侧选股漏斗。本轮没有改左侧基本价值线，也没有改买点、加仓、止盈、止损、仓位和交易执行。

## 边界

多氟多、天赐材料、均胜电子只用于验证买卖思想，不用于硬调选股。

正确用法：

```text
把样本放进股票池。
检查同一套买卖规则在样本上的买点、加仓、止盈、止损是否接近白大思想。
如果买卖节奏明显不符，说明买卖解释要复盘。
如果买卖节奏基本相符，只能说明买卖规则解释基本通过；不能反推选股漏斗必须选出这些股票。
```

右侧选股不再沿用“结构接近分、均线趋势分、量价节点分”作为核心门槛。它们可以保留为诊断字段，但专业右侧漏斗主要按横截面量化因子排序。

## 当前右侧逻辑

第一步，硬门槛：

```text
质量分 >= 70
净利润同比 >= 10%
市值 >= 100 亿
必须有可用日线和成交量
```

第二步，横截面多因子排序：

```text
质量因子：质量分越高越好
成长因子：净利润同比越高越好，但做上限截断，避免低基数爆表
流动性因子：市值 + 20 日平均成交额代理
中期动量因子：剔除最近 5/20 日后的 60 日、120 日趋势
相对强度因子：60 日、120 日涨幅和距离 120 日高点
趋势稳定因子：60 日上涨天数比例、20 日均线斜率、60 日均线斜率
低风险因子：60 日回撤、20 日波动、60 日下行波动
不过热因子：5 日涨幅不过分极端，20 日不追过热
主线交集加分：新鲜主线快照里有归属，额外加分
```

第三步，风险经理否决：

```text
普通入池：
    排名 <= 90
    综合分 >= 66
    20 日平均成交额代理 >= 5 亿
    中期动量排名 >= 68
    趋势稳定排名 >= 52
    低风险排名 >= 30
    短期不过热排名 >= 30
    20 日涨幅 >= 0
    60 日回撤 >= -32%

强趋势例外：
    排名 <= 60
    综合分 >= 70
    20 日平均成交额代理 >= 10 亿
    中期动量排名 >= 78
    趋势稳定排名 >= 50
    低风险排名 >= 25
    短期不过热排名 >= 22
    60 日回撤 >= -38%
```

第四步，组合经理排序：

```text
quant_right 可以获得右侧交易权限。
quant_right 不再进入每日核心保底名额。
每日核心保底仍只给 value_model 和 standard_mainline。
quant_right 只能靠量化分与其他候选竞争剩余名额。
```

第五步，交易执行：

```text
候选池只决定 allow_right。
真正买点仍由统一买卖引擎判断。
止盈、止损、尾仓、加仓、左侧网格全部保持原样。
```

## TradingAgents 参考

参考 TauricResearch/TradingAgents 后，本轮没有照搬外部代码，而是吸收它的流程思想：

```text
牛方研究员：寻找强动量、相对强度、主线交集和高流动性。
熊方研究员：否决高回撤、过热、低稳定和无主线归属的弱趋势票。
风险经理：限制 quant_right 硬占核心名额。
组合经理：让左侧价值线、标准主线、右侧量化在同一候选接口里竞争。
```

## 回测结果

严格交易参数不变：

```text
最大右侧持仓数：3
最多持有股票数：5
最小买点证据分：7
止盈分仓：5 份
尾仓最低收益：50%
不启用支撑拉回首仓试错
收盘确认按 close_proxy
交易成本、印花税、滑点保持原设置
```

阶段结果：

```text
v6 专业均衡漏斗：
    2024-09-24 至 2026-07-14：-5.239%
    2026 年以来：+31.951%

v7 动量进攻漏斗：
    2024-09-24 至 2026-07-14：+24.766%
    2026 年以来：+11.900%

v10 风险经理漏斗（当前代码）：

    结论：不保留，候选记忆导致交易过度。

    2026 年以来：+28.925%

    2024-09-24 至 2026-07-14：+64.816%
```

当前没有达到“2024-09-24 至今 200%+、今年 30%+”的目标。继续强调参可以制造更好样本内结果，但容易变成后验拟合。

## 数据清洗和 Alpha 因子

本轮新增了候选侧数据血缘和清洗字段：

```text
data_status：traded / invalid_price_bar / suspended_or_no_trade
tradestatus：数据源交易状态，若缓存提供则使用
is_traded_bar：价格有效且交易状态允许
raw_amount：数据源原始成交额，若缓存提供则使用
amount_source：raw / close_x_volume_proxy / missing
price_source：行情来源
valid_price_bar：OHLCV 基本合法性
```

成交额口径：

```text
优先使用 raw_amount。
raw_amount 缺失时才用 close * volume 代理。
代理成交额会写 amount_source，不能再当成无血缘的真实成交额。
```

新增 Alpha 风格量价字段：

```text
alpha_volume_price_corr_20：20 日量价相关
alpha_turnover_expansion_20：短期量能相对 20 日扩张
alpha_close_position_60：收盘在 60 日高低区间的位置
alpha_close_position_120：收盘在 120 日高低区间的位置
alpha_intraday_strength_20：20 日日内收盘强弱
alpha_gap_5d_count：5 日跳空次数
quant_alpha_rank：上述量价 Alpha 的横截面综合排名
```

当前最好版本额外加入了“均衡回踩候选”：

```text
排名 <= 85
综合分 >= 64
20 日平均成交额代理或真实额 >= 10 亿
中期动量排名 >= 58
Alpha 量价排名 >= 45
低风险排名 >= 60
短期不过热排名 >= 55
20 日涨幅 >= 4%
60 日回撤 >= -16%
```

这个通道不是买点，也不是加仓规则，只是让“不是最强动量、但成交额足、回撤浅、不过热”的干净回踩票进入候选竞争。

## 当前判断

买卖引擎不是主要矛盾，但候选池仍不够专业：

```text
右侧强趋势能选到一部分，例如中国巨石、华工科技、兆易创新。
但候选漏斗对 2024-2025 大牛阶段的主线轮动捕捉不够。
过度收紧可以降低回撤，却会错过今年收益。
过度进攻可以改善长周期，却会放大今年回撤。
```

## 数据缺口

需要补全的数据：

```text
严格时点财务：当前 financial_point_in_time=False，缺公告日期和历史修订版本。
真实成交额、换手率、自由流通市值：当前 avg_amount_20 = close * volume，只是代理。
行业和主题历史成分：当前主线快照可用，但不足以做严格行业中性。
指数和行业基准：缺沪深 300、创业板、行业指数相对强度。
ST、停牌、涨跌停、上市天数：右侧风控还不够显式。
盈利修订、ROE、毛利率、现金流、负债率、估值分位：当前只有聚合质量分和同比。
交易事件血缘：部分右侧卖出事件没有完整保留原始 candidate_source 和 signal_type。
```

下一轮更应该补数据和做训练/验证切分，而不是继续用同一段样本强调阈值。
