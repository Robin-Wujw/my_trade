# 回测盈利改进方案 - 2026-07-15

## 一、现状分析

### 1.1 当前回测表现

**基线对比**（2024-09-24 至 2026-07-14）：
- **历史基线**（dual_ledger_final）：+114.063%，最大回撤-19.977%
- **最新回测**：+0.04%，最大回撤-16.86%
- **收益落差**：~114个百分点

**主要亏损来源**：
1. **中信博价值网格**：质量分、扣非增长、市值三重失效，2025-05-07清仓造成约-25,000元总亏损
2. **右侧止损频繁**：新易盛、华峰测控、乐鑫科技等多次-2%~-6%止损退出
3. **左侧价值标的选择**：焦作万方当前浮亏-21.86%

### 1.2 核心问题识别

#### 问题1：财务失效预警机制不足
- 中信博从2024-09-25建仓到2025-05-07证伪清仓，期间约7个月
- 质量分、扣非增长、市值同时失效说明基本面恶化早有迹象
- **改进点**：需要更早的财务预警和渐进退出机制

#### 问题2：trade_basis_score买点验证不够精准
- 从流水看，很多"33未三日下行首仓"、"连阳波段一半拉回"等买点快速止损
- 说明这些"合格买点"实际上买点质量不高
- **改进点**：需要对照IMA和博主网页端文章，优化买点打分权重

#### 问题3：止损参数可能过于严格
- 多个标的在-2%~-3%就止损，但可能只是正常波动
- 例如艾力斯多次-2%~-3%止损后，最后一份却+30%止盈
- **改进点**：需要根据不同结构类型调整止损空间

#### 问题4：左侧价值标的筛选标准
- 焦作万方浮亏-21.86%说明价值线可能高估或财务质量不足
- **改进点**：提高左侧标的质量门槛，增加安全边际

## 三、具体改进措施

### 3.1 【高优先级】优化左侧价值网格财务预警

**问题**：中信博案例显示7个月才触发证伪清仓，期间损失巨大。

**改进方案**：

1. **增加月度财务健康检查**
   ```python
   # 建议在候选生成阶段增加
   - 质量分连续2个月下降超过5分 → 预警
   - 扣非增长连续2个季度下滑 → 预警  
   - ROE低于10%且持续下降 → 预警
   - 总市值跌破120亿元（留20%缓冲） → 预警
   ```

2. **渐进退出而非一次性清仓**
   ```python
   # 预警后的处理
   - 第一次预警：停止新增左侧买入，保留已有网格
   - 第二次预警（隔月再次触发）：卖出50%仓位
   - 第三次预警或明确证伪：全部清仓
   ```

3. **价值线动态调整**
   - 每季度财报后重新计算价值线
   - 价值线下修超过10%时，网格价位同步下调
   - 避免"守株待兔"式的固定价位

**实施文件**：
- `stock_research/strategies/fundamental_selection.py` - 增加财务预警函数
- `stock_research/strategies/portfolio_backtest.py` - 修改左侧网格退出逻辑

### 3.2 【高优先级】重新标定trade_basis_score权重

**问题**：当前很多8分以上的"合格买点"快速止损，说明打分权重可能不准确。

**改进方案**：

1. **使用IMA工具验证权重**
   ```python
   # 当前trade_basis_score最高12分，建议调整为：
   - MA20/MA60同时上扬：3分（当前可能过低）
   - 收盘站上MA20：2分
   - 贴近MA20（2%以内）：+1分
   - 距21日高点3%以内：1分（降权，因为单纯接近高点不是买点）
   - 5/10日量能确认：3分（提高权重）
   - 均量扣抵改善：2分
   ```

2. **增加"禁止买入"的负面特征**
   ```python
   # 即使总分达标，以下情况也不买入：
   - MA20在MA60下方 → 禁止
   - 当日收盘低于MA20超过3% → 禁止
   - 5/10日量能基准一个都不满足 → 禁止
   - 近5日连续缩量 → 禁止
   ```

3. **回测历史成功案例提取特征**
   ```python
   # 从艾力斯、华峰测控等最终盈利案例中提取：
   - 成功买点通常具备哪些共同特征？
   - 快速止损的买点缺少哪些关键要素？
   - 建立"成功买点特征库"用于打分优化
   ```

**实施文件**：
- `stock_research/strategies/fundamental_selection.py` - 调整trade_basis_score计算
- 新增 `stock_research/analysis/entry_point_analysis.py` - 买点成功率分析工具

### 3.3 【中优先级】优化止损空间和条件

**问题**：多个标的在-2%~-3%就止损，可能过于敏感。

**改进方案**：

1. **根据结构类型差异化止损**
   ```python
   # 当前统一5%~10%，建议调整为：
   - W底颈线突破：止损5%（颈线位置明确）
   - 跳空长阳：止损3%（缺口下沿明确）
   - 平台突破：止损6%（需要给波动空间）
   - U50/R50突破：止损7%（趋势延续需要空间）
   - 量价节点：止损5%（节点明确）
   - 连阳一半/支撑拉回：止损8%（回调正常）
   ```

2. **增加"缓冲期"机制**
   ```python
   # 买入后前3个交易日内：
   - 不触发时间止损
   - 空间止损放宽2%（给适应期）
   - 除非跌破硬止损位（如跳空缺口）
   ```

3. **根据市场环境调整**
   ```python
   # Formula33阶段影响：
   - 明确上行阶段：止损可放宽1%~2%
   - 明确下行阶段：止损收紧1%
   - 波动率VIX高时：止损适当放宽
   ```

**实施文件**：
- `stock_research/strategies/portfolio_backtest.py` - 修改stop_loss计算逻辑

### 3.4 【中优先级】提高左侧价值标的质量门槛

**问题**：焦作万方浮亏-21.86%，说明价值线筛选标准需要加强。

**改进方案**：

1. **提高质量分要求**
   ```python
   # 当前左侧候选：质量分>=50
   # 建议调整为：
   - 自动价值网格：质量分>=75（更严格）
   - ROE>=12%（当前可能没有此要求）
   - 负债率<60%
   - 经营现金流/净利润>=0.8
   ```

2. **增加安全边际**
   ```python
   # 当前：0.80 <= 现价/价值线 <= 1.08
   # 建议调整为：
   - 首次建仓：0.70 <= 现价/价值线 <= 0.95（更保守）
   - 网格上方：可以到1.00
   - 左转右阈值保持1.08不变
   ```

3. **行业和周期性筛选**
   ```python
   # 增加行业排除清单：
   - 强周期行业（有色、煤炭等）质量分要求>=85
   - 轻资产行业需要额外验证现金流
   - 避免单一客户依赖度>30%的公司
   ```

**实施文件**：
- `stock_research/strategies/fundamental_selection.py` - value_model候选筛选
- 配置文件增加行业权重和排除清单

### 3.5 【低优先级】优化Formula33阶段判断

**问题**：当前33上下行判断可能过于机械。

**改进方案**：

1. **增加市场情绪指标**
   ```python
   # 除了21日去重数ΔN，还可参考：
   - 涨停板数量变化
   - 板块轮动速度
   - 两市成交额变化
   ```

2. **动态调整试错仓位**
   ```python
   # 当前：下行30%/上行20%
   # 可以更细化：
   - 刚进入下行（1-2日）：25%
   - 明确下行（3日+）：20%
   - 刚进入上行（1-2日）：25%
   - 明确上行（3日+）：30%
   ```
