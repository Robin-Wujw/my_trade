# 观察日与时点规则

## 1. 基本定义

- **观察日**：本次分析站在的自然日；
- **行情截止日**：观察日及以前最后一个可用交易日；
- **报告期**：财务报表覆盖的会计期末，不等于披露日；
- **实际数据日**：某份行情或板块数据真正覆盖到的最后交易日。

各生产步骤通过显式日期参数、缓存元数据、产物日期校验和日报门禁维护时点边界。

## 2. Formula33 日期窗口

Formula33 先从交易日历选择行情截止日，再取最近 21 个交易日。指定 `--start-date` 和 `--end-date` 时只保留区间内交易日。

2026-06-11 至 2026-07-10 是 21 日固定回归区间。自然日周末请求映射到最近交易日；如果该交易日的相同参数已经完整成功，完成清单直接复用并输出 `network_fetch=0`。

指标计算前必须满足：

```text
trade_date <= 行情截止日
```

前复权日线可以包含为指标预热所需的更早历史，但不能包含截止日之后的数据参与计算。

## 3. 观察日交易资格

窗口内技术命中和观察日是否可交易是两个状态：

- 观察日存在有效日线：`traded`；
- 最近有效日线早于观察日交易截止日，且接口能够确认无交易：`suspended_or_no_trade`；
- 请求失败、空响应或历史不足：`data_unavailable`。

正式 Formula33 名单只保留 `traded`。无交易股票保留在技术全量和诊断中；数据不可用股票失败关闭，不能用“停牌”解释接口错误。资格只对本次观察日有效，下一观察日重新判断。

## 4. 上市天数与市值

上市天数按观察日和上市日期计算，正式技术全量要求超过 300 天。总市值大于 100 亿元是独立参考池，不是 188 口径 Formula33 的基础条件。

市值缺失不能自动视为大于阈值。市值独立池必须记录实际来源和覆盖率。

## 5. 财务数据

生产脚本按明确 `report_period` 读取或增量补齐财务缓存，并把报告期、观察日、来源和抓取时间写入截面或元数据。基本面筛选的技术价格日期不得晚于观察日。

当前财务文件缓存没有完整保存每次公告和修订版本，也没有统一财务时点数据库。因此：

- 当前每日生产可以使用截至观察日选择的报告期；
- 缺少可信实际披露时间的数据不能声称满足严格历史时点回测；
- 历史回测不得用今天缓存的最新修订版冒充历史观察日版本；
- 法定披露截止日只能作为保守边界，不能伪造逐股实际披露日。

## 6. 板块数据

板块步骤保存实际最后交易日和来源。板块行情必须满足当前步骤要求的新鲜度、最少历史行数和覆盖率，过期或缺失不能静默参与主线评分。

历史日期请求如果使用当前板块成分，元数据必须标记为 `unsafe`，不能把它当成严格历史成分。当前生产观察使用当期板块和当期成分。

## 7. 日报输入门禁

日报使用本轮明确捕获的三个必需输入：

1. Formula33 CSV；
2. 板块观察 CSV；
3. 基本面筛选 CSV。

三者的观察日必须一致。样例文件、缺失文件、同类型多个本轮文件或完成清单版本不匹配都会阻止日报。六个上游步骤没有全部成功时不得进入正式推送。

## 8. 回测边界

- 所有指标先按观察日截断，再计算；
- 财报只能使用观察日当时已经公开的版本；
- 板块行情和成分都要有当时有效来源；
- 名单冻结前不能读取未来收益；
- 缺少披露版本或历史成分证据时必须标记降级，不能宣称严格无未来数据。
