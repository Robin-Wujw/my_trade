# 结果输出阅读指南

## 先看什么

每日运行完成后按以下顺序查看：

1. 打开最新 `formula33_stats_*.xlsx`，先读“先看这里”。
2. 查看“近21日正式名单”：这是观察日仍可交易的正式技术去重池。
3. 查看“33公式日统计”：判断数量是在扩大还是收缩。
4. 查看“观察日状态”：确认停牌、数据不可用和行情来源状态。
5. 再看每日综合 HTML 中的基本价值线、正常基本面候选和主流板块。

“近21日技术命中”“停牌技术命中诊断”都是解释差异的诊断表，不等于当天可以买入。

## Formula33 工作簿

| 工作表 | 用途 | 是否正式结果 |
|---|---|---|
| 先看这里 | 阅读顺序、公式和状态说明 | 说明页 |
| 33公式日统计 | 每日 BASE/XG 数量及近21日趋势 | 市场结构判断 |
| 横向统计 | 将日期横向排列，便于观察连续变化 | 市场结构判断 |
| 命中股票 | 每个交易日的 BASE/XG 原始记录 | 明细，不去重 |
| 近21日正式名单 | 近21日曾满足XG且观察日仍可交易 | 是 |
| 近21日技术命中 | 只按技术条件统计，包含观察日停牌股票 | 否，诊断 |
| 市值大于100亿池 | 正式技术池中的大市值参考子集 | 参考池 |
| 停牌技术命中诊断 | 技术满足但观察日停牌或无交易 | 否，诊断 |
| 观察日状态 | 每只股票的最新行情日、状态、来源和异常 | 数据验收 |

### BASE 和 XG

- `BASE`：某日同时满足 KDJ-K > 80、WR10/WR20 < 20、RSI9 > 70、上市超过 300 天。
- `XG`：BASE 连续 5 个交易日成立，是正式技术信号。
- “近21日正式名单”：在近21日中至少出现一次 XG，并且观察日仍可交易，按股票去重。

### 常用统计列

| 中文列 | 含义 | 怎么看 |
|---|---|---|
| 当日BASE数量 | 当天同时满足四项基础条件的股票记录数 | 只反映当天强势面宽度 |
| 当日XG数量 | 当天连续5日满足BASE的股票数 | 比BASE更严格 |
| 近21日正式可交易去重数 | 期间XG股票去重后，排除观察日停牌/不可用 | 正式技术池规模 |
| 近21日技术命中去重数 | 不检查观察日交易状态 | 用于解释被排除数量 |
| 窗口趋势斜率 | 近21日去重数量变化方向 | 正数偏扩散，负数偏收缩 |
| 数据不可用数量 | 行情接口失败或历史不足 | 非零时先处理数据问题 |

### 观察日状态

- 可交易：行情覆盖观察日，可进入正式池。
- 观察日停牌或无交易：只进入诊断，不进入正式池。
- 数据不可用：接口失败、返回不完整或历史不足；正式任务不会把它静默当成停牌。
- 完整 CSV 缓存命中：请求区间已在本地完整缓存，不需要联网补行情。
- DuckDB 或网络补齐：本次需要从数据库恢复或向数据源补缺口。

## CSV 与 Excel 的区别

- CSV 面向程序，继续使用稳定英文字段，避免破坏日报和历史回归。
- Excel 面向人工，使用中文表头、中文状态和“先看这里”说明页。
- 不要手工修改 CSV 后再让流水线继续运行；人工备注应放在单独文件。

## 控制台进度

Formula33 每 200 只或任务结束时输出：

```text
行情进度 1200/5599 | 成功 1180 | 完整缓存 900 | 跳过 15 | 失败 5 | 剩余 4399 | BASE/XG记录 326
```

- 成功：已获得足够历史行情并完成计算。
- 完整缓存：成功股票中直接命中完整 CSV 缓存的数量。
- 跳过：上市天数不足等不具备计算资格的股票。
- 失败：数据不可用，需要查看“观察日状态”的异常说明。
- BASE/XG记录：信号明细行数，不是去重股票数。

## 最终决策边界

Formula33 是市场强势结构和候选池，不是单独的买入指令。实际阅读顺序仍应是：技术池 → 基本面与价值线 → 主流板块 → 流动性与风险 → 人工决策。

## PushPlus 手机推送

PushPlus 不固定条数。系统按完整名单和字符上限自动分页，每页独立校验长度，全部页面合并后再校验股票代码完整性。任一分页发送失败都视为本次推送失败，可只补发失败页，不能把部分成功误报为全部成功。

阅读顺序：

1. 两个月突破强提醒：只在存在 45%～60% 回调修复股票时置顶。
2. 市场阶段、数据完整性和价值线候选。
3. 正常基本面候选、主流板块和风险边界。

完整名单按“可操作性从高到低、同分时风险从低到高”排列。表格使用自然语言：

- `上涨量能达标`：今日量同时高于 5 日均量、10 日均量、5 日扣抵量和 10 日扣抵量；
- `量能接近达标/一般/不足`：直接写明还有几项未满足，不显示难懂的 `3/4`；
- `暂无指标警讯`：KD、RSI、MACD 没有需要展开的异常；
- `KD开口≥20`、顶背离或底背离：只有实际触发时才标红；
- `上涨支撑价`：完整上涨波段的 50% 价位；
- `回调突破价`：当前有效回调的 50% 价位，并显示现价距离；
- `回调修复`：当前回调低点到前高的进度，严格用于当前有效回调。

技术评分中的“可操作性”和“风险”是多指标归一化结果，用于排序和快速检查，不是收益概率。KD、RSI999、MACD、ENE、WR、BIAS、基准量比值及各项原始值保留在 CSV，手机推送默认只展开异常。

两个月突破追踪池独立于当天筛选：最近约 45 个交易日曾进入前两组选股的股票都会保留。回调 45%～50% 为强提醒区；重新上穿 50% 会累计第几次突破，50%～60% 继续跟踪；超过 60% 停止提醒。只有突破前高完成并且脱离当前全部筛选，才从追踪池删除。

若 PE/PB 方法存在 `valuation_percentile`，它表示当前估值在自身历史样本中的位置，与价格波段进度不能混用。价值线股票显示“价值线的 X 倍”，不是买入指令。

每次日报运行都会额外生成 `var/exports/reports/pushplus_preview_*.html`。该文件内容与实际各条推送相同，中间仅增加分隔线，适合发送前在浏览器检查。
