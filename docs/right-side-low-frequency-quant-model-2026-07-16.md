# 右侧低频量化选股模型更新（2026-07-16）

## 外部研究结论

本次没有直接复刻任何机构模型。幻方量化公开资料只说明其方向是 AI 驱动、深度学习平台和高性能研究基础设施，并未公开具体股票打分公式或持仓规则：

- High-Flyer 官网写明其定位包含 `AI Driven Hedge Fund`，并介绍 Fire-Flyer II 深度学习平台和面向研究训练的算力系统：<https://www.high-flyer.cn/en/>

所以本项目不能写成“幻方公式”，只能吸收公开量化研究的共性原则：

1. Gu、Kelly、Xiu 的机器学习资产定价研究显示，收益预测里更有效的信号集中在价格趋势、流动性和波动率；树模型和神经网络优势来自非线性和因子交互，但也强调样本外和防过拟合：<https://academic.oup.com/rfs/article/33/5/2223/5758276>
2. Qlib 强调完整量化研究链路：数据、监督学习、市场动态建模、组合和执行，并提供 `Alpha158/Alpha360` 等多特征数据集：<https://github.com/microsoft/qlib>
3. MASTER 提出市场引导的特征选择和跨股票关系建模，核心启发是：因子有效性随市场状态变化，不能用固定单一动量分数硬排：<https://arxiv.org/abs/2312.15235>
4. TRA 用多个交易模式和路由器处理市场里不同形态的样本，启发是：强趋势、回调修复、高盈亏比不能强行塞进同一个入口：<https://arxiv.org/abs/2106.12950>
5. DoubleEnsemble 用样本重加权和特征选择降低金融数据低信噪比下的过拟合，启发是：候选漏斗要惩罚缺失数据和不稳定因子，不能只追求回测收益：<https://arxiv.org/abs/2010.01265>

## 本项目低频化解释

我们不做高频，不做订单簿，不预测下一分钟。目标是：

```text
基本面硬门槛不变
→ 右侧候选用横截面多因子排序
→ 找趋势已经证明、但买点仍有盈亏比的股票
→ 继续交给白大买卖逻辑做建仓、加仓、止盈、止损
```

因此模型不是“涨最多就买”，而是加入四类低频约束：

1. 趋势效率：上涨是否用较小回撤和较低下行波动换来。
2. 盈亏比代理：不是越贴近高点越好，而是强趋势中有受控回撤、结构靠近、止损空间可定义。
3. 量价确认：放量是否伴随上涨，是否存在有效量价节点，而不是孤立放量。
4. 过热控制：短期跳空、5日急涨、20日过热会降权。

## 新增因子

候选快照新增字段：

- `quant_trend_efficiency_rank`：趋势效率排名。
- `quant_payoff_rank`：低频盈亏比代理排名。
- `quant_structure_rank`：结构位置排名。
- `quant_volume_confirm_rank`：量价确认排名。

原字段继续保留：

- `quant_momentum_rank`
- `quant_low_risk_rank`
- `quant_trend_stability_rank`
- `quant_overheat_control_rank`
- `quant_alpha_rank`
- `quant_divergence_rank`

## 入选路径

右侧量化候选现在有三条路径：

```text
标准量化：
  综合分 >= 68
  成交额真实存在且 >= 5亿
  动量、趋势稳定、低风险、不过热、盈亏比代理均达标

强趋势：
  综合分 >= 72
  成交额 >= 10亿
  动量极强
  允许低风险分稍低，但仍要求最低盈亏比代理

高盈亏比：
  综合分 >= 66
  成交额 >= 10亿
  动量不必最高
  但低风险、结构位置、量价确认、盈亏比代理必须更强
```

缺失 `avg_amount_20` 不再默认当成高流动性。缺成交额就是数据不合格，不能因为模型想选它而放行。

## 与白大思想的关系

这次只改“右侧选股漏斗”，不改左侧价值线，不改建仓、加仓、止盈、止损。

买卖仍然要求：

```text
候选入池
→ 出现可解释结构买点
→ 仓位按 Formula33 阶段和已有浮盈解锁
→ 每笔绑定止损
→ 盈利后按原分仓止盈体系处理
```

右侧模型只回答“哪些股票值得等买点”，不直接下单。
