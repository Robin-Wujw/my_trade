# 策略回测与止损止盈修改摘要（v9）

## 已完成

- 50%、62.5%、75%定义为价格结构比例，不再与仓位绑定。
- 单只股票所有交易批次合计上限为62.5%；自动左侧观察仓入口已移除。
- 所有选股模型使用统一候选接口，观察名单不能注入候选；每日通过统一门槛和排名的前10只进入同一买卖模型。
- 普通A股按100股一手、`688`科创板按200股一手执行买卖。
- 50%、62.5%、75%结构价位保留且默认启用。显式配置锚点优先；自动锚点只有通过时间窗口和MA共振检查后才允许产生条件单，可用 `--no-auto-price-structure` 做关闭敏感性测试。
- 21日突破改为纯价格条件单，不使用当天最终成交量决定当天盘中成交或候选排序。
- 候选排序优先基本面评分，再比较技术信号和此前已知量能。
- 新增批次取得10%或20%安全垫前，禁止同股继续加仓；安全垫只使用前一交易日已知收盘价。
- 保留最大浮盈回撤一半、峰值回撤10%、成本上方5%和时间条件止盈。
- 收盘条件支持两种撮合：默认14:55/收盘代理，另提供次日开盘保守敏感性口径。
- 加入佣金、卖出印花税和估算滑点；修复汰弱换强评分条件短路。
- 候选快照和Formula33默认下一交易日生效。

## 交叉验证后的边界

IMA知识库明确指出，比例位本身没有支撑性，必须与均线、量价节点、连阳一半等真实技术支撑重叠。回调波段50%突破还要求对应级别回调过程中曾跌破62.5%，否则容易只是频繁假突破的盘整平台。

因此当前把上涨波段拉回、上涨U50趋势恢复与回调波段突破分成三种结构。生产默认使用结构价位，显式锚点优先，自动锚点必须通过时间与均线共振过滤。U62.5/U75只做拉回；上涨U50有效跌破后重新向上突破可作为独立趋势恢复信号：

```json
{
  "price_structures": [{
    "kind": "uptrend_support",
    "uptrend_low": 38.4,
    "uptrend_high": 61.44,
    "ratio": 0.625,
    "confluence": ["MA60", "volume_node"]
  }]
}
```

回调波段一半突破必须使用三锚点，并通过“上涨波段62.5%曾被跌破、50%价格乘1.1小于62.5%价格、整理至少13个交易日”三项检查：

```json
{
  "price_structures": [{
    "kind": "pullback_recovery",
    "uptrend_low": 28.61,
    "uptrend_high": 61.44,
    "pullback_low": 38.4,
    "consolidation_days": 15
  }]
}
```

## 数据刷新门禁

组合回测默认先刷新全市场K线到请求截止日，再按“候选交易日历 → 同日主线快照 → 最终候选快照 → Formula33阶段历史”的顺序重建全部派生输入。请求日遇周末或休市日时，以Formula33完成清单确认的最近交易日作为统一有效截止日。候选和Formula33末日不一致时直接终止，不允许悄悄混用旧缓存。

只有显式传入 `--no-refresh-inputs` 才使用冻结数据，终端和结果摘要都会标记 `inputs_refreshed=false`；该模式只用于同一份输入下的策略对比，不代表最新生产数据。

## 候选依据增强（2026-07-14）

统一候选模型在质量、增长、市值、主线和价值线之外，新增 `trade_basis_score`。该分数只使用观察日收盘前已经可见的信息：MA20/MA60 是否上扬、收盘是否站上或贴近 MA20、是否距离 21 日收盘高点 3% 以内、量能是否高于 5/10 日基准、5/10/20 日均量扣抵是否呈现多周期改善。

这些字段会写入每日候选 CSV 和 `derived.candidate_snapshots`：

```text
trade_basis_score
trade_basis_reason
technical_alignment
ima_web_validation
validation_sources_json
```

IMA 与网页对照只作为证据审计：比例位、均线和均量必须与真实结构或量能条件互相印证，不直接绕过统一买卖模型生成交易。成交事件会保留当时的 `selection_reason`、`trade_basis_reason` 和 `technical_alignment`，最终中文流水会显示为更直观的买入/卖出动作摘要。

## 多级结构锚点v14

旧自动算法“最近500日最低点—之后最高点”已删除。新算法按以下顺序识别：

1. 使用左右各3根K线确认上涨/下降拐点，只有右侧确认完成的拐点才可用于当天以前的计划；
2. 找涨幅至少2%且成交量不低于5日、10日前参照量的放量启动日；
3. 为每个启动日同时保留最近确认低点（小级别）和近期最低确认低点（趋势级别）；
4. 从启动日向后确定阶段高点H，形成多组L-H上涨结构，不再压成唯一全局结构；
5. U75/U625/U50只有与前一日已知上扬均线接近时才生成拉回单；
6. 上涨U50有效收盘跌破后的重新突破作为独立趋势恢复买点；
7. 回调R50使用H-P三锚点，仍要求同一L-H的U625先被跌破、幅度检查通过且整理至少13日。
