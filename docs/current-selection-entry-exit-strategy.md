# 当前选股策略（完全版）与买卖策略（详细版）

本文是当前代码口径的总说明，优先级高于旧研究报告、参数扫表和临时诊断文档。当前最终链路是：

```text
MiniQMT/本地行情与财务缓存 -> 严格时点候选快照 v4 -> MiniQMT 回测执行画像 -> 组合买卖引擎
```

## 1. 不变量

- MiniQMT 只作为只读数据源和回测执行画像使用，实盘下单 API 仍然禁用。
- 所有候选只使用观察日当时可见的信息。行情切到观察日，财务使用逐公司公告日可见口径；不允许用回测结束后的财报、复权锚点或行业名单倒灌历史。
- 候选池只负责“选哪只”；买卖引擎只负责“何时买、买多少、错了哪里止损、盈利怎么处理”。选股强不等于立刻买入。
- 人工观察名单、持仓计划和历史牛股复盘不能直接注入候选池，只能作为显式计划或诊断。
- 买入信号默认次日生效；MiniQMT 回测里用 `close_proxy` 近似 14:55/收盘确认执行。日 K 回测不能声称自己知道盘中真实排队成交。

## 2. 最终候选池

候选快照由 `stock_research/strategies/historical_candidates.py` 生成，版本为 `unified-selection-v4`。最终回测使用：

- 候选池：`var/backtests/pit_candidates_20230103_20260714_final/manifest.json`
- 回测摘要：`var/backtests/miniqmt_pit_final_20230103_20260714/miniqmt_2023-01-03_2026-07-14_summary.json`
- 口径：`strict_financial_point_in_time=true`，`unsafe_snapshot_count=0`

### 2.1 硬门槛

所有可交易候选先过同一组基本门槛：

- `quality_score >= 70`
- `earnings_yoy >= 10%`
- 观察日可见总市值 `mktcap >= 100` 亿元
- 观察日有有效价格、成交量和成交额
- 停牌、无交易、缺关键财务或缺行情的股票不补假数据

这组门槛是防止选股模型为了收益去“捡不该买的票”。右侧强势股也不能绕过质量、成长和市值底线。

### 2.2 旧模型降级为诊断

旧的左侧价值线、普通主线和“领导力”叙事不再拥有候选保留名额，也不再决定最终入选。它们只保留三个用途：

- 作为审计字段解释某只股票是否曾被旧思路覆盖；
- 作为人工复盘材料，帮助发现模型漏掉的产业事实；
- 作为买卖引擎里的兼容输入，避免旧交易计划和单测失效。

主线板块标签仍可用于同行业/同主题持仓约束，但不再给 `right_quant_score` 加分。价值线也只作为诊断和人工计划参考，不再自动挤占最终候选池。

### 2.3 多因子量化候选

最终执行候选只来自 `candidate_source=factor_quant`。模型先在通过硬门槛的股票中做时点可见的横截面因子排序，再按不同风险收益形态入选。

主要因子：

- 质量、成长、流动性：`quality_score`、`earnings_yoy`、市值、20 日均成交额
- 动量与相对强度：5/20/60/120 日收益、距 120 日高点、60 日剔除近 5 日动量、120 日剔除近 20 日动量
- 趋势效率：60 日收益与波动、下行波动、回撤之间的比值
- 低风险：60 日回撤、20 日波动、60 日下行波动
- 趋势稳定：60 日上涨天数比例、MA20/MA60 斜率
- 结构位置：21 日区间收敛、收盘在区间上沿、有效量价节点数量、离量价节点距离
- 量价确认：收益与量能相关、换手/成交扩张、有效节点、日内强弱
- 过热控制：短期涨幅过快、跳空次数、回撤修复过急会扣分
- 背离环境：KD/RSI/WR 顶背离扣分，底背离加观察价值
- 市场状态：按横截面 20/60 日收益中位数和上涨比例分为进攻、防守、平衡，并给不同因子加权

入选路径：

- `standard_quant`：排名、综合分、流动性、动量、结构、低风险、回撤全部达标
- `strong_trend`：强趋势例外，要求排名更靠前、成交额更高，并限制短期追高
- `high_payoff`：强调盈亏比代理，要求低风险、结构、量价和回撤更优
- `asymmetric_pivot_watch`：适合刚从平台/枢轴启动的右侧股
- `compact_attack_core`：最终模型最重要的核心攻击形态，要求排名前 50、`right_quant_score >= 85`、结构排名 >= 80、量价确认 >= 60、低风险 >= 60、60 日收益 >= 15%、60 日回撤不差于 -8%

所有入选路径统一标记为 `candidate_source=factor_quant`。`right_quant_setup` 只解释形态类型，不决定候选配额。

### 2.4 每日排序

候选综合分不是买入分，而是候选排序分：

```text
candidate_score = right_quant_score
```

候选按 `candidate_score` 统一排序，不再保留价值/主线核心名额。候选池上限只是观察宽度，避免买点出现时因为名单太窄而漏掉；是否买入仍交给买卖引擎的结构、盈亏比、止损距离、仓位和组合约束。

## 3. 买入策略

当前买入不再靠“7 分、8 分”这类旧证据分硬卡。`entry_evidence_score` 只作为解释字段，真正的门是“语义高盈亏比买入门”，代码字段为 `semantic_high_reward_risk`。

### 3.1 买入信号类型

买入引擎只接受可解释的右侧结构：

- `consolidation_breakout`：窄幅整理平台收盘/触发突破
- `leader_pivot_breakout`：强势股枢轴突破
- `gap_long_ma_breakout`：跳空长阳越过中长期均线
- `uptrend_50_reclaim`：上涨波段 50% 支撑位重新收复
- `pullback_50_breakout`：回调波段 50% 突破确认
- `volume_price_node`：有效量价节点形成后再确认
- `bull_run_half_pullback`：连阳后一半回踩再转强
- `uptrend_support_pullback`：上升趋势支撑拉回；高质量候选允许 15% 到 20% 小仓首仓试错，已有强右仓时也可作为加仓结构
- `leader_trend_add`：已证明强趋势后的追加结构

无结构、纯盘口触碰、只因为进入候选 Top10、只因为单日上涨，都不能买。

### 3.2 语义高盈亏比买入门

候选有完整画像时，必须同时满足：

- 质量、成长、市值硬门槛仍然成立
- `trade_basis_score >= 6.5`；弱量价基础不买
- 20 日均成交额通常不低于 5 亿元；`compact_attack_core` 可放宽到 4.5 亿元
- 突破类信号的观察日量能确认 `known_volume_ratio >= 1.0`
- 不追后段加速：20 日收益超过 70%，或 20 日收益超过 45% 且量能放大到 3 倍以上，拒绝
- 止损必须有效：止损价必须低于入场价，且能解释失效位置
- 风险距离：普通结构不超过 10%，支撑拉回不超过 6.5%；Formula33 连续下行达到 3 日时，上限收紧到 7%
- 计划盈亏比：普通结构 `reward/risk >= 2.5`，支撑拉回 `>= 3.0`；市场连续下行达到 3 日时也至少要求 3.0
- 目标价来自可见结构：枢轴延续、前高、平台高点、波段锚点等，不用未来最高价倒算

这就是“先知道错在哪，再决定值不值得赌”的部分，也是最接近白大高盈亏比买卖法的核心。

### 3.3 下单时点

- 候选在观察日收盘后确定。
- 默认 `signals_effective_next_day=true`：买入信号次日才可执行。
- 回测里 `close_proxy` 用收盘价近似 14:55 确认；真实 MiniQMT 可以在当日盘中挂条件单，但回测若没有分钟级数据，就不能把盘中触发当成已知事实。
- 条件单、限价单或计划单不成交时记录 `order_not_filled`，不强行按收盘补成交。

### 3.4 仓位

当前仓位按白大网页原文重新校正，默认不再使用 50%/60% 首仓。白大侧网页交叉验证依据包括：

- 2026-05-13：右侧交易在基本面族群里等技术面突破或拉回再试错，右侧出手最多三成，一次一只，没止损或没浮盈超过 10% 前不开第二只新仓。
- 2026-04-13：Formula33 未确认上行 3 天前，试错仓不宜直接三成，1.5 到 2 成都可以；进入上行后才正常三成。
- 2026-04-01：右侧初始三成，后续靠浮盈加仓做到 4.5 成、6.75 成，而不是首笔半仓重压。
- 2026-07-14：初始 2 成时，浮盈 10% 后可找机会加到 3 成；已有 3 成时，需要等 20% 浮盈再继续加。

落地规则：

- 主交易目标最多 3 只
- 全部持仓股票硬上限 5 只
- 同行业或高相关主题最多 2 只
- 单股最大暴露由回测参数控制，当前默认 70%，用于允许浮盈加仓后的趋势仓发展，不代表首仓可满打
- 标准右侧首仓上限 30%
- Formula33 未确认上行时，试错首仓优先 15% 到 20%
- Formula33 连续下行达到 3 日时，首仓降到 20%
- 支撑拉回首仓默认允许小仓试错：合格形态 15%，极强形态 20%，并配 7% 空间止损和 8 日时间止损；Formula33 连续下行达到 3 日时仍禁止拉回首仓
- 已有右仓加仓前，旧批次必须已经证明过；拉回加仓最多按现有仓位 50%，其它加仓最多按 1/3
- 左转右时，右侧新增仓不超过原左侧仓位的一半

被废弃的激进变体：此前“高确定性首仓 50%”“非对称枢轴首仓 60%”只能解释为量化回测激进参数，不符合白大默认仓位纪律，已从当前默认口径移除。

## 4. 卖出策略

卖出分为防守卖出、时间卖出、分仓止盈、弱换强四类。

### 4.1 防守止损

每笔右侧买入都带独立结构止损：

- 止损来自买点结构，不是固定亏损百分比硬套
- 入场后若次日确认失败，触发再确认失败退出
- 新批次未证明前保留自己的买点止损
- 支撑拉回试错批次额外有 7% 空间止损
- 止损价不允许高于或等于买入价；这种信号直接拒绝

### 4.2 时间止损

买入后如果迟迟不证明自己，就退出或降级：

- 普通右侧批次需要在有限时间内出现足够最大浮盈
- 支撑拉回试错批次 8 日内不证明则退出
- 连续下行市场里对证明要求更高
- 时间止损只使用此前交易日已经确认的信息，不用当日收盘倒推盘中资格

### 4.3 分仓止盈

当前 MiniQMT 回测程序默认 `profit_tranches=5`，通用组合函数支持 2 到 5 份。默认不再使用 2 份止盈，因为它过于粗糙，容易把白大“多条件、多份额、留强去弱”的分仓思想压成两刀卖出。

白大侧网页交叉验证依据包括：

- 2026-03-11：分仓止盈用于在主升段中保留大部分仓位，同时用多个卖点保护利润；卖点包括新高后 13 日不创新高、背离后 5 日不放量不创新高、跌破最近量价节点、跌破上涨常态前低、最大浮盈回撤一半。
- 2026-05-26：案例中使用多份止盈，对不同技术弱化证据分别卖出。
- 2026-06-12：浮盈超过 10% 后可设成本线上方 5% 的条件单；浮盈超过 20% 后可用 10% 分仓止盈其中一份。

止盈原则：

- 最大浮盈不到 10% 时不主动分仓止盈，优先让正确的交易证明自己
- 达到 10%/20% 等盈利保护条件后，按配置份数分批退出
- 最后一份强制保留，只有最大浮盈回撤一半等尾仓条件才最终退出
- 背离、跌破上扬 MA20、有效量价节点失效等最多各卖一份，不是机械清仓表
- 新加仓批次取得 10% 到 20% 浮盈后才和底仓合并；合并后不重置已执行的止盈层级

### 4.4 弱换强

只有新机会明显更好时才考虑弱换强：

- 新机会必须 `semantic_entry_ok=true`
- 新机会盈亏比至少 5，风险距离不超过 6%
- 旧持仓通常要跌破 MA20、持仓至少 5 日且收益不到 10%
- 新旧评分差必须够大；如果旧标的已被候选池剔除，替换门槛略低
- 卖出和买入按执行时点拆开记录，不用未来结果证明替换正确

### 4.5 成本与撮合

MiniQMT 回测画像：

- 佣金率：`0.000085`
- 最低佣金：5 元
- 卖出印花税：`0.0005`
- 估算滑点：`0.0005`
- A 股按整手撮合
- T+1 和可用份额由持仓批次逻辑约束
- 停牌、涨跌停、订单未触发会记录阻断原因

## 5. 与白大标准的符合度

符合的部分：

- 选股与买卖已分离：强候选不自动等于买入
- 买入前必须回答“为什么是它、为什么是现在、错了在哪里止损”
- 买点以右侧结构、量价确认和盈亏比为核心
- 止损是保护性结构止损，不是亏损后临时放宽
- 盈利仓允许分批止盈和尾仓跟随，不急着把强趋势卖光
- 候选和回测使用严格时点数据，不允许未来函数
- MiniQMT 只读，当前不会自动实盘下单
- 首仓上限已从激进的 50%/60% 改回 30%，仓位放大只允许通过浮盈证明后的加仓完成
- 默认止盈份数已从 2 份改回 5 份，保留多条件分仓和最后趋势尾仓

仍需人工确认的部分：

- 日 K 回测里的 `close_proxy` 只能近似 14:55/收盘确认，不能证明真实盘中挂单成交质量
- `compact_attack_core` 是量化抽象，不是白大原话；它把平台收敛、量价确认、低回撤、趋势效率转成了可回测条件
- 板块/主题标签仍不如人工产业理解细，同行业限制可能偏粗
- 当前文档说明的是策略和回测口径，不构成实盘建议
