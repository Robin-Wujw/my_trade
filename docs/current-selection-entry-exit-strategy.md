# 当前选股、建仓、加仓、止损与止盈策略

> 版本：2026-07-15。本文是当前自动回测的统一执行口径；历史研究文档中的旧版“21日高点直接买入”“右侧最多3只”“右侧最多4只”“左侧同时2只”等描述不再覆盖本文。所有判断只使用决策日当时可见的数据。本文用于研究和程序验收，不构成投资建议。

## 1. 决策链

```text
时点财务与产业门槛
→ 每日候选池与排序
→ 同行业/高相关暴露检查
→ 分配左侧权限、右侧权限或双权限
→ 左侧价值网格 / 右侧技术主结构识别
→ 量能、节点、扣抵、大波段多证据打分
→ Formula33决定20%或30%首仓
→ 每个新增批次绑定自己的止损
→ 浮盈后在新合理买点加仓
→ 已证明批次合并并分仓止盈
→ 高收益最后一份按最大浮盈回撤一半退出
```

选股回答“今天优先看谁”，技术信号回答“今天能不能买”。基本面或领导力分再高，也不能替代技术买点门槛。

## 2. 数据和时点约束

- 财务报告只能从法定可见日后参与历史选择，价格、财务期和观察日必须一致。
- 行情使用单一提供方的一条前复权序列，不能拼接不同提供方历史。
- 波段拐点必须经过右侧K线确认；当前交易日不能用于证明此前并不知道的锚点。
- 候选和Formula33在收盘后形成时，从下一交易日生效。
- 冻结本地数据运行必须明确标记 `--no-refresh-inputs`，不能伪装成最新在线数据。

## 3. 选股硬门槛

正常基本面、主流和强势成长候选统一要求：

- 质量分 `>= 70`；
- 扣非同比增长 `>= 10%`；
- 总市值 `>= 100亿元`；
- 当日存在有效价格和成交量。

总市值门槛是左右侧自动新买入的共同入口条件。模型候选只要缺少观察日可见总市值，或总市值低于100亿元，即使价格形态或价值线接近也不得开新左仓或新右仓；已有持仓仍按原批次止盈、止损、价值证伪和组合限额处理。

基本价值线候选还要求前复权收盘价/价值线位于 `0.80～1.08`。价值线提供左侧网格资格，不直接产生右侧订单；只有同时来自主流/强势成长模型，或已有左仓后出现独立技术信号，才有右侧权限。若扣非同比 `>= 100%`、总市值 `< 150亿元` 且收盘价/价值线 `> 0.90`，视为高景气弹性但左侧安全边际不足，只保留诊断，不给自动左侧建仓权限。

主流板块身份增加排序权重，但不能绕过财务硬条件。行业数据缺失、过期或使用当前成分代理历史时必须在结果中披露。

## 4. 候选来源与排序

每日候选最多10只，至少保留5个价值线或主流基本面核心名额，其余位置由全体合格候选竞争。

### 4.1 核心分

```text
核心分 = 质量分
       + min(max(扣非增长, 0), 100%) × 20
       + 主流板块加分15
       + 价值线接近度0～5
       + trade_basis_score 0～12
```

`trade_basis_score`只用于候选排序，来自MA20/MA60方向、收盘相对MA20、距21日收盘高点、5/10日量能和5/10/20日均量扣抵。它不产生订单。

### 4.2 强势成长领导力

强势成长观察要求 `trade_basis_score >= 4` 且 `leadership_score >= 15`。领导力只使用观察日可见的20/60/120日涨幅和距120日高点位置，最高30分。

```text
最终候选分 = 核心分 + leadership_score
```

### 4.3 买点内排序

同一天多个候选都达到技术执行门槛时：

```text
入场优先级 = 候选分 + 技术证据分 × 8 + min(已知量比, 5) × 2
```

该排序只决定先后；不满足技术最低分的股票不会因候选分高而买入。

### 4.4 左右权限不是互斥分类

| 候选来源 | 左侧权限 | 右侧权限 |
|---|---|---|
| 仅 `value_model` | 有 | 初始没有；已有左仓后可按独立技术信号左转右 |
| 仅 `standard_mainline` / `growth_leadership` | 没有 | 有 |
| 价值模型与主流/成长模型同时命中 | 有 | 有 |

同一只高质量股票可以同时具备左侧和右侧权限，但执行上不允许左侧网格与右侧趋势仓并行运转。左仓涨离价值线（收盘价/价值线 > 1.08）并出现独立合格右侧信号时，原左侧批次切换为右侧管理，绑定本次右侧结构止损和止盈规则，左侧网格锚暂停；右侧周期清完以前，不再按网格卖出或重新开左仓。新左仓严格按当日统一候选分排序竞争名额，不能按文件顺序或股票代码顺序抢占仓位。

## 5. 持仓和相关性限制

- 主交易名额默认3只；参数敏感性测试最多可放到5只，但任何口径下全部持仓标的硬上限都是5只，日常目标仍以3只左右为宜。
- 所有行情下左侧价值标的最多1只；左转右后该标的不再占左侧名额，但在右侧周期结束前也不能重新开左侧网格。若已有多只左侧标的，只保留“已有右仓优先、候选分更高、持仓暴露更大”的一只，其余按组合限额在下一交易日开盘退出；这是钛弱留强和组合限额执行，不是价值证伪，也不使用日榜落选作为理由。
- 同行业或相近产业链最多2只。
- 优先使用当日可见的板块标签；标签缺失或过细时，以入场前60个交易日收益相关性补充，至少40个有效样本且相关系数 `>= 0.60` 视为相近暴露。
- 中际旭创、新易盛、天孚通信、胜宏科技等虽非同一个申万细分行业，仍可能因高收益相关或共同算力硬件标签计入相近产业暴露。
- 仅当最后一份当前浮盈仍 `>= 50%` 时，才可成为不占主仓名额的高收益尾仓；普通低收益尾仓仍占名额。

## 6. 技术买点与证据分

生产回测最低技术证据分为8。扣抵、量能、均线状态和节点只是证据；没有主结构时不能独立下单。单一结构通常不足以下单，必须叠加位置、趋势、量能或扣抵等多维证据。

### 6.1 主结构基础分

| 主结构 | 基础分 | 成交确认 | 批次止损锚 |
|---|---:|---|---|
| 跳空长阳越过MA20/MA60 | 5 | 向上缺口、阳线实体至少3%、量能确认 | 缺口下沿 |
| 整理平台突破 | 4 | 至少21日窄幅整理，区间振幅不超15%，收盘放量突破平台上沿 | 平台上沿 |
| U50跌破后收复 | 5 | 收盘重新站上U50，成交量不低于5/10日较高基准 | U50 |
| 回调波段R50突破 | 5 | U62.5曾跌破、幅度合格、整理至少13日，收盘放量突破 | R50 |
| 有效量价节点 | 3 | 节点支撑上浮2%～4%内收盘确认 | 节点两日低点 |
| 连阳一半拉回 | 3 | 至少3根连阳后拉回一半位置 | 连阳一半 |
| U50/U62.5/U75支撑拉回 | 3 | 比例位与已知均线/结构共振，盘中拉回成交 | 对应比例位 |

单纯盘中触及21日高点、收盘突破21日收盘高点、盘中短暂越过U50/R50、单纯接近某条扣抵均线都不买。整理平台可以使用21日以上区间作为识别窗口，但买点来自“窄幅平台”这个主结构，而不是21日高点本身。距21日高点和领导力只能用于候选排序或观察，不能作为独立买入主结构。

### 6.2 证据加分

- 当日量能不低于5/10日较高基准：`+1`。
- 当日量能达到5/10日较高基准的1.3倍：额外 `+1`。
- 收盘买点必须处于结构价上方5%以内；5%只是合格区域，不自动加分。距离结构价2%以内才加 `+1`。
- 买点贴近上扬MA20、回踩上扬MA20后收回，或从MA20下方有效收复：`+2`；收在上扬或刚收复的MA20上方再加 `+1`。
- MA20位于MA60上方，且买入前一日MA20/MA60均较5日前上扬：`+2`。
- 量价节点处于明确上行趋势：`+1`。
- 最近60日存在至少3个尚未失效节点：`+2`。
- MA20/MA60/MA120低价低量扣抵共振：分别 `+1/+2/+3`。
- 主结构对应上涨幅度至少50%的大波段：`+2`。
- 向上跳空：`+1`。

扣抵共振还要求结构价距离均线不超过5%、均线上扬、被扣价格低于均线至少5%、被扣量低于近60日中位量的80%，且当前成交量不低于被扣量。

### 6.3 证据扣分

- 买入参考价高于结构价5%：`-2`，不再视为理想结构区。
- 结构止损距离超过7%：`-1`；超过10%：`-2`。
- 股价低于MA20且MA20低于MA60：`-2`。

### 6.4 有效和无效量价节点

候选节点采用可审计近似：当日成交量至少为近60日中位量1.5倍且涨幅至少2%。

1. 形成当日不可使用。
2. 下一交易日后，以形成两日最低价为支撑。
3. 后续任一交易日最低价跌破支撑，旧节点永久失效。
4. 跌破后重新站回不能复活旧节点，只能等待新节点或其他新结构。
5. 事件必须记录节点日、确认日、支撑价和当时有效节点数量。

该定义是对原文规则的日K量化近似；不同市值和股性对应1%～4%上浮幅度仍待自适应，当前统一使用2%～4%。

## 7. 左侧价值网格

- 网格资金分10份，当前每份账户资金2%，单股左仓通常上限20%、硬上限不超过30%。
- 价值线首次成交5份：3份价值核心、2份上方交易份；格距不得低于5%。
- 两个上方交易份分别在价值线上方一格、两格卖出；下方5份逐格买入，并在回到上一格时卖出。
- 已卖网格可以在以后重新跌回原买价时买回；同一根日K既触发卖出又覆盖买价时不做有利的日内往返假设。
- 日常候选榜落选不等于价值逻辑失效。已经启动的网格保留原价值锚并继续运行；只有新的时点财务证据明确给出质量门槛失效、扣非增长失效、市值/流动性硬门槛失效、价值线实质下修或候选失败原因为 `value_financial_falsification` 时，才终止整个左侧计划并在下一交易日开盘清仓。未入选但缺少明确失败原因时，只停止新增左侧买入，不清掉既有网格。
- 纯价值候选先涨离价值线并出现独立合格右侧信号时，左侧网格整体切换为右侧管理，首个新增右侧批次不超过当时左仓的一半；原左侧批次不再挂网格卖单，而是跟随右侧止损、分仓止盈和尾仓规则。切换时原 `L` 批次重新分配为右侧 `R` 批次，原 `L` 编号只作为审计来源保留，不再出现在后续卖出信号中。右侧清仓后，只有重新跌回左侧区间、仍被当前候选选中且价值逻辑未证伪，才允许重新建立左侧网格。若候选同时拥有价值和主流/成长权限，右侧可按正常首仓规则执行，但同样会先接管既有左侧批次。

## 8. 右侧首仓与新仓解锁

- Formula33连续下行达到3日：合格买点首仓20%。
- 连续下行不足3日，包括方向未确认：合格买点首仓30%。
- 第一只主仓历史最大浮盈达到过20%后才解锁第二只；连续上行达到3日时，已有主仓历史最大浮盈达到过10%即可继续解锁。新仓解锁看“是否证明过”，不是只看当前回撤后的浮盈；加仓仍要求当前浮盈满足门槛。
- Formula33连续上行达到3日时，已有主仓均达到10%即可继续解锁；否则要求最大浮盈至少20%、其余已有主仓至少10%。
- 账户总名义仓位不超过100%，单股合并仓位不超过62.5%。

生产口径下，上涨波段 U50/U62.5/U75 支撑拉回主要用于已有右仓后的加仓，不作为普通首仓。为了复盘“能不能更早抓到强趋势牛股”，组合回测提供 `--allow-pullback-pilot` 敏感性模式：仅当 Formula33 未三日下行、右侧市场仍可试错、候选 `leadership_score >= 24`、`trade_basis_score >= 6`、证据分至少10分时，允许 `uptrend_support_pullback` 作为“领先族群拉回试错首仓”。该首仓只用15%，绑定原结构止损，同时额外使用7%空间止损和8个交易日时间止损。该模式不绕过总持仓、行业/相关主题、整手现金和汰弱留强规则。

2026-01-01 至 2026-07-14 冻结回测显示：默认生产版最终收益 `+0.440%`、最大回撤 `-17.177%`；打开 `--allow-pullback-pilot` 后，右侧买入从20笔增至29笔，但最终收益降至 `-3.636%`、最大回撤 `-14.673%`；再把主交易名额放到5只，最终收益进一步降至 `-8.738%`、最大回撤 `-17.567%`。因此该模式用于诊断和人工复盘，不纳入默认生产策略。W底自动识别已禁用，避免把高位趋势中继误判为筑底。

## 9. 浮盈加仓

加仓必须同时满足：第一笔已取得规定浮盈、出现新的独立合理买点、现有未证明批次为零。

- Formula33下行达到3日时，新增批次需原仓浮盈至少20%；其他阶段至少10%。
- 拉回买点最多加已有仓位的1/2。
- 突破买点最多加已有仓位的1/3；高位新高可进一步降到1/4。
- 每个新增批次保存独立成本、日期、结构止损和费用。
- 新批次取得10%～20%安全垫后才与核心仓合并；合并不重置已经执行的止盈层级和峰值观察。

这对应“1成→加0.5成→再加0.75成→再加1.12成”的思想，但实际仓位还受账户现金、整手和单股上限约束。

## 10. 止损

### 10.1 入场阶段

每个新批次执行三类止损：

- 条件止损：收盘跌破该批次的U50、R50、节点、颈线、平台、均线或缺口锚。
- 幅度止损：买入前设定5%～10%，10%为极限，盘中触发按OHLC撮合。
- 时间止损：弱市5～8日、中性至良好最多13日，仍未取得约10%浮盈则退出。

“用什么进，就用什么止损”只约束尚在入场验证阶段的独立批次，不能把所有批次混成平均成本后统一放宽。

### 10.2 原始结构止损何时仍有效

普通核心仓即使已经执行过部分止盈，仍保留最初买点的结构止损；如果盈利没有发展成高收益尾仓，回到原结构以下就应退出，而不是因为曾经盈利过就无限持有。只有同时满足“已减到最后一份、当前浮盈仍至少50%”的高收益尾仓，才进入只减不加的尾仓管理；普通仓后续再次入选并出现新买点时，应建立新一轮批次，而不是依赖旧仓占位。

尚未证明、尚未合并的新加仓批次始终使用各自入场止损。

未发生左转右的左侧价值核心不使用右侧的结构止损、空间止损或时间止损。若同一股票先有独立右侧仓，且左侧网格仍未切换，两套计划才按各自规则处理；但一旦触发左转右，原左侧批次已经转为右侧批次，后续卖出必须使用右侧止损、分仓止盈和尾仓规则，不得继续按左侧网格卖。

## 11. 分仓止盈和最后一份

分仓止盈独立于Formula33方向。

- 最大浮盈低于10%：不启动分仓，只执行入场止损。
- 最大浮盈10%～20%：仅允许最大浮盈回撤一半保护一份。
- 最大浮盈超过20%：启用利润底线、峰值回撤、背离计时、上扬MA20有效跌破、最近有效量价节点跌破等中间条件。每个不同节点只管理一份；节点跌破后永久失效。
- 最后一份只允许“最大浮盈回撤一半”最终卖出；MA20跌破或峰值回撤15%不能单独清掉最后一份。

当前目标份数可配置2～5，默认5；实际份数还要受持股整手数量约束。例如普通A股只持有200股时自动按2份处理，不会因为总仓低于10%就在第一个止盈条件整批清仓；科创板仅持有一个200股买入单位时只能形成1份。每个中间触发最多执行一份，并永久为最终条件保留最后一份：

| 配置 | 中间份额 | 最后一份 |
|---|---|---|
| 3份 | 每次卖约1/3，最多执行2个中间条件 | 仅最大浮盈回撤一半 |
| 5份 | 每次卖约1/5，最多执行4个中间条件 | 仅最大浮盈回撤一半 |

当实际只有1份时，不执行中间分仓条件：未达到高收益尾仓标准前仍受原始结构止损约束，达到最后一份且当前浮盈至少50%后，才只按最大浮盈回撤一半退出。重新创新高会重置不过高计时；量能恢复或背离钝化会重新评估相应中间条件。

每笔止盈事件必须在流水中记录实际触发ID（利润底线、峰值回撤、背离计时、有效节点跌破或 `maximum_profit_half`），不能只写“统一分仓止盈”，以便区分最后一份究竟因何退出。

## 12. 作者案例验收标准

## 13. 案例不一致时如何处理

案例验收失败时按以下顺序归因：

1. 复权口径是否一致；
2. 作者使用最高价/最低价还是最高收盘/最低收盘；
3. 当日是否已经能确认拐点、节点或财务信息；
4. Formula33阶段和仓位是否一致；
5. 是否把左侧网格、右侧试错和左转右混成一种操作；
6. 若以上都一致仍失败，才判定程序规则偏离并修改代码。

不能为了让某个案例通过而使用未来数据、事后选择锚点或为单只股票写例外。

## 14. 牛股漏选复盘

中际旭创、新易盛、长飞光纤、工业富联、洛阳钼业这类强趋势样本不能用来反向硬编码个股例外，但必须用来检查规则是否把“合理右侧”卡得过死。复盘按以下漏斗输出每个关键日期的失败层级：

1. 候选层：是否通过质量、扣非增长、100亿元市值、主流/强势成长或价值权限。
2. 权限层：仅价值候选是否被正确限制为左侧，已有左仓是否满足左转右条件。
3. 主结构层：是否出现跳空长阳、窄幅平台、U50收复、R50突破、量价节点或连阳一半，而不是只有MA、扣抵或21日高点。W底暂不作为自动买点。
4. 证据层：7分门槛差几分，缺的是量能、2%结构距离、MA20、MA20/MA60、扣抵、大波段还是被风险扣分。
5. 组合层：是否被第二只解锁、同行业/相关主题最多2只、总持仓5只、左侧最多1只或现金整手约束拦截。
6. 换仓层：若名额已满，新票是否明显强于弱票，旧票是否同时满足跌破MA20、持有满5日、浮盈不足10%等弱势条件。

调整顺序固定为先补结构识别和证据记录，再评估分数权重；不得取消5%结构区或放松行业/仓位上限来追历史收益。2024-09-24至2026-07-14的18组评分 sweep 显示，单纯降低门槛不优，较优组合是把连阳一半拉回和已有右仓后的上涨波段比例支撑拉回基础分从3提到4，同时把生产执行门槛从8调到7。
